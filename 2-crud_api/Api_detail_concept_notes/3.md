Here is a **clear client vs server cheat sheet** for API communication (short and structured).

---

# 1️⃣ HTTP Methods (Client → Server request type)

| Method     | Used For                 | Example             |
| ---------- | ------------------------ | ------------------- |
| **GET**    | Fetch data               | get students        |
| **POST**   | Create data              | add student         |
| **PUT**    | Replace/update full data | update whole record |
| **PATCH**  | Update partial data      | change only name    |
| **DELETE** | Remove data              | delete student      |

Example (client):

```python
requests.get(URL)
requests.post(URL, json=data)
requests.patch(URL, json=data)
requests.delete(URL)
```

---

# 2️⃣ Client-Side Parameters (`requests` library)

| Parameter  | Meaning                             | Example       |
| ---------- | ----------------------------------- | ------------- |
| `params=`  | Data sent in **URL query**          | `?id=1`       |
| `data=`    | **Form body data**                  | HTML form     |
| `json=`    | **JSON body** (most APIs)           | API data      |
| `.json()`  | Convert response JSON → Python dict | read response |
| `headers=` | Send extra request info             | auth token    |

Example:

```python
r = requests.get(URL, params={"id":1})
r = requests.post(URL, json={"name":"A"})
data = r.json()
```

---

# 3️⃣ Server-Side Request Objects (Django / DRF)

| Thing            | Purpose                               |
| ---------------- | ------------------------------------- |
| `request.method` | Which HTTP method client used         |
| `request.GET`    | Query parameters                      |
| `request.POST`   | Form data                             |
| `request.body`   | Raw body                              |
| `request.data`   | DRF parsed data (JSON/form/multipart) |

Example:

```python
if request.method == "POST":
    data = request.data
```

---

# 4️⃣ Serializer (Server)

| Thing                   | Meaning                     |
| ----------------------- | --------------------------- |
| `Serializer()`          | convert model ↔ python data |
| `serializer.data`       | serialized output           |
| `serializer.is_valid()` | validate input              |
| `serializer.save()`     | save to database            |

Example:

```python
serializer = StudentSerializer(data=request.data)
serializer.is_valid()
serializer.save()
```

---

# 5️⃣ Response Tools (Server)

| Tool             | Purpose                  |
| ---------------- | ------------------------ |
| `Response()`     | DRF response object      |
| `HttpResponse()` | Django basic response    |
| `JSONRenderer`   | convert Python → JSON    |
| `content_type`   | tells client data format |

Example:

```python
return HttpResponse(json_data, content_type="application/json")
```

or DRF:

```python
return Response(serializer.data)
```

---

# 6️⃣ Render / Convert Data (Server)

| Tool           | Converts              |
| -------------- | --------------------- |
| `JSONRenderer` | Python → JSON bytes   |
| `JSONParser`   | JSON → Python         |
| `BytesIO`      | in-memory file stream |

---

# 7️⃣ Headers / Metadata

| Header          | Meaning                  |
| --------------- | ------------------------ |
| `Content-Type`  | format of body data      |
| `Authorization` | authentication           |
| `Accept`        | response format expected |

Example:

```
Content-Type: application/json
```

---

# 8️⃣ Data Flow (Full API flow)

```
CLIENT
requests.post(json=data)
        ↓
HTTP Request
        ↓
SERVER
request.data
        ↓
Serializer
        ↓
Database
        ↓
serializer.data
        ↓
Response / JSONRenderer
        ↓
CLIENT
r.json()
```

---

✅ **One rule to remember**

| Side   | Purpose            |
| ------ | ------------------ |
| Client | send request       |
| Server | process + database |
| Client | read response      |
